# Brand assets — design spec

**Date:** 2026-05-31
**Scope:** Propagate the canonical v4 tracelane and peek brand marks across all
product surfaces that currently use placeholder or missing assets.

---

## Problem

The v4 brand marks exist in a separate portfolio directory but are not in the repo.
Five surfaces ship with placeholder or absent branding:

| Surface | Current state |
|---|---|
| Peek extension toolbar icons (5 PNG sizes) | Solid dark-blue fill — no mark |
| Tracelane docs `favicon.svg` | Generic line-chart, wrong color |
| Peek docs `favicon.svg` | Generic circle-dot, wrong color |
| Tracelane HTML report hero | No logo anywhere |
| All product READMEs + root README | No logo header |

---

## Brand marks (source)

From `blog-article/portfolio/company/v4/`:

**`sub-tracelane.svg`** — 256×256 viewBox, `#C2563D` stroke, transparent background.
Shape: a horizontal rounded-rect (trace bar) with three dashed connector segments
and a filled circle endpoint. Represents a pipeline/trace lane.

**`sub-peek.svg`** — 256×256 viewBox, `#4A6B82` stroke, transparent background.
Shape: an eye (lens path + iris circle + filled pupil). Represents observation/capture.

---

## Approach: A — Copy source SVGs, generate PNGs at build time

Single source of truth in `assets/brand/`. Static SVG surfaces updated directly.
Extension PNGs generated by a one-shot script. No new runtime packages.

---

## Section 1 — Brand asset source

Copy both marks into the repo:

```
assets/
  brand/
    sub-tracelane.svg
    sub-peek.svg
```

These are committed static files. READMEs reference them via raw GitHub URLs.
All other surfaces embed or generate from local copies.

## Section 2 — Favicon updates

Both docs-site favicons keep the existing format (colored rounded-rect background +
white mark inside, 64×64 SVG) but replace the inner graphic with the v4 marks.

**`apps/tracelane-docs/public/favicon.svg`**
- Background rect: `#C2563D`, `rx="12"`, 64×64
- Inner mark: trace-bar shape (rounded rect + three dashed segments + circle endpoint)
  scaled and re-stroked in white to fit within ~44×18 px centered in the 64×64 canvas

**`apps/peek-docs/public/favicon.svg`**
- Background rect: `#4A6B82`, `rx="12"`, 64×64
- Inner mark: eye shape (lens path + iris + pupil) scaled and re-stroked in white
  to fit within ~40×28 px centered in the 64×64 canvas
- Color shifts from current `#1e6fd9` to `#4A6B82` to match the v4 canonical color

Both files are self-contained SVG — no external dependencies.

## Section 3 — Extension icon generation

**Script:** `packages/peek-extension/scripts/generate-icons.mjs`

Logic:
1. For each target size `[16, 32, 48, 96, 128]`:
   - Build an icon SVG string from scratch (not by reading `assets/brand/sub-peek.svg`
     directly — the source mark uses `#4A6B82` on transparent, but the icon needs
     white marks on a `#4A6B82` background, so the paths are re-coloured inline):
     `#4A6B82` rounded-rect background + white peek eye mark scaled with padding
     proportional to size
   - Use `sharp(Buffer.from(svgString)).png().toFile(...)` to render PNG
   - Write to `packages/peek-extension/public/icon/<size>.png`

**Dependency:** `sharp` added to `devDependencies` in
`packages/peek-extension/package.json`.

**NPM script** added to the same `package.json`:
```json
"generate:icons": "node scripts/generate-icons.mjs"
```

Run with: `pnpm --filter @peekdev/extension generate:icons`

Not wired into the `build` script — icons are stable committed assets, regenerated
only when brand changes. Generated PNGs replace the existing placeholders in-place;
`wxt.config.ts` and the manifest are unchanged.

## Section 4 — Tracelane report hero logo

**File:** `packages/tracelane-report/src/metadata.ts` → `renderHero` function

The tracelane mark is added as the leftmost item in the `.eyebrow` flex row, before
the pulsing status dot.

Implementation: inline SVG literal embedded in the `renderHero` return string.
The report is a self-contained offline HTML file — `<img src="...">` would break
offline mode, so inline SVG is the only correct approach.

The SVG is a reduced version of `sub-tracelane.svg`, rescaled for `height="16"` with
strokes proportionally thickened for legibility at small sizes. Color: `#C2563D`.

No CSS changes needed — the eyebrow is already `display: flex; align-items: center`
and accepts new children.

## Section 5 — README headers

A logo `<img>` is added as the first line of each README, above the `# heading`,
using raw GitHub URLs to the committed `assets/brand/` SVGs.

**Root `README.md`** — both marks:
```md
<img src="https://raw.githubusercontent.com/Cubenest/rrweb-stack/main/assets/brand/sub-tracelane.svg" height="40" alt="tracelane">&nbsp;&nbsp;<img src="https://raw.githubusercontent.com/Cubenest/rrweb-stack/main/assets/brand/sub-peek.svg" height="40" alt="peek">
```

**tracelane package READMEs** (`tracelane-wdio`, `tracelane-core`, `tracelane-report`,
`tracelane-cli`) — tracelane mark only, `height="40"`.

**peek package READMEs** (`peek-mcp`, `peek-cli`, `peek-extension`) — peek mark only,
`height="40"`.

**`rrweb-core` README** — no logo (internal package, not a product surface).

Height `40` renders well on GitHub in both light and dark mode. SVGs have transparent
backgrounds so no light/dark mode handling needed.
